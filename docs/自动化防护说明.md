# ShellWard 自动化防护说明

减少人为操作，尽量自动化的防护能力。

## 已实现的自动化

| 能力 | 触发时机 | 说明 |
|------|----------|------|
| **OpenClaw 漏洞检查** | 每次启动 | 对比漏洞库，若当前版本受影响则 logger 告警 |
| **插件风险扫描** | 每次启动 | 快速扫描 extensions/plugins 中的高危模式（eval、反弹 shell、数据外泄） |
| **MCP 配置扫描** | 每次启动 | 检测 mcp.json、settings.json 中的可疑 URL、命令 |
| **root 运行告警** | 每次启动 | 若以 root 运行则告警 |
| **ShellWard 更新检查** | 每次启动 | 24h 内最多检查一次，发现新版本则提示 |

### 配置

```json
{
  "autoCheckOnStartup": true   // 默认 true，设为 false 可关闭启动时自动检查
}
```

## 定时检查（cron）

用于每日/每周定期检查，无需启动 OpenClaw：

```bash
# 1. 添加 crontab
crontab -e

# 2. 添加一行（每天 9:00 执行）
0 9 * * * /path/to/shellward/scripts/cron-security-check.sh >> ~/.openclaw/shellward/cron.log 2>&1
```

`cron-security-check.sh` 会：
- 运行 `standalone-check.js` 检查 OpenClaw 版本与漏洞
- 统计审计日志中的 CRITICAL/block 数量
- 输出 OpenClaw 版本

## 半自动化（一键操作）

| 项目 | 命令 | 说明 |
|------|------|------|
| **升级 OpenClaw** | `/upgrade-openclaw yes` | 一键升级到最新版，无需手敲命令 |
| **环境隔离** | `/harden` | 输出 Dockerfile 模板，复制即可构建安全容器 |
| **网络限制** | `/harden` | 输出 iptables 防火墙规则，复制即可应用 |
| **恶意插件移除** | `/scan-plugins` | 发现风险插件后输出 `openclaw plugins uninstall xxx` 命令，复制执行 |

## 仍需人工判断的部分

| 项目 | 原因 |
|------|------|
| **升级 OpenClaw** | 大版本升级前需确认插件兼容性 |
| **环境隔离** | 首次部署容器需用户配置 `.env` 和端口映射 |
| **防火墙规则** | 需确认不影响其他服务再执行 |
| **恶意插件** | 误报时需用户判断是否确实移除 |

## 总结

- **启动时**：自动检查漏洞、插件、MCP、root，发现问题即告警
- **定时**：cron 脚本可每日检查，结果写入日志
- **一键操作**：升级、隔离、防火墙、卸载插件均提供复制即用的命令/模板
- **人工判断**：仅大版本升级兼容性、首次容器部署、防火墙副作用需人工确认
